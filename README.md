## 1. Project Overview
This repository contains the technical component of the Capstone Project for the OSS course[: 4]. The project involves a structured audit of the **Linux Kernel**, focusing on its origin, GPL v2 licensing, and its functional footprint within a Linux ecosystem[: 12, 13, 24].

## 2. Repository Structure
The repository is organized as follows:
* `scripts/`: Contains five functional shell scripts demonstrating Linux automation and philosophy[: 14, 89].
* `manifesto_anshu.txt`: The output generated by Script 5[: 198].
* `README.md`: Project documentation and execution guide[: 225].

## 3. Shell Scripts Gallery
Each script includes detailed comments and adheres to the project requirements[: 90, 92].

| Script | Name | Description | Key Concepts  |
| :--- | :--- | :--- | :--- |
| **Script 1** | `system_identity.sh` | Displays kernel version, uptime, and OS license[: 94]. | Variables, Command Substitution. |
| ****Script 2** | `package_inspector.sh` | Checks for kernel package installation and details[: 125]. | `if-then-else`, `case` statements. |
| ****Script 3** | `directory_auditor.sh` | Audits permissions and sizes of `/boot` and `/lib/modules`[: 145]. | `for` loops, `awk`, `du`. |
| ****Script 4** | `kernel_log_analyzer.sh` | Analyzes kernel logs for errors or warnings[: 163]. | `while-read` loops, Positional params. |
| ****Script 5** | `manifesto_generator.sh` | Generates a personalized FOSS manifesto[: 185]. | User input, File redirection. |

## 4. How to Run
1.  **Clone the repository:**
    ```bash
    git clone [Your Repository URL]
    cd OSS-AUDIT
    ```
2.  **Provide execution permissions:**
    ```bash
    chmod +x scripts/*.sh
    ```
3.  **Execute a script (Example - Script 1):**
    ```bash
    ./scripts/system_identity.sh
    ```

## 5. Philosophical Note
This project explores the essence of open source: software that is built openly, shared freely, and improved collectively[: 10]. By auditing the Linux Kernel, this work highlights the importance of the GPL v2 license in protecting the "four freedoms" of software.
